integração de limite de crédito

A integração de status de cliente é de origem distribuidor para plataforma, e para esta ação, será disponibilizado pela Infracommerce, o endereço do endpoint e credenciais.

Conceito: Uma vez que um determinado CNPJ possua limite de crédito, status de aprovação é liberado na plataforma a funcionalidade que permite de compras em forma de boleto faturado. É fundamental que o distribuidor gerencie os limites e disponibilize esses valores apenas para os clientes desejados e que tenham confiabilidade no mercado, a fim de evitar inadimplência.

Exemplo de requisição de limite de crédito





customerDocumentNr = CNPJ do cliente que será concedido limite.

customerDocumentNr = CNPJ do cliente que será concedido limite.

Origin = SELLER (Limite gerenciado pelo distribuidor). Caso esse campo seja informado com valor:IFC e seja enviado com qualquer valor fictício, o limite de crédito será gerenciado pela Infracommerce. Esse tipo de integração gerenciada pela plataforma vale apenas para casos para projetos gerenciados pelo Infrapay. Em caso de dúvidas, falar com o time de projetos da infracommerce para mais informações.

Informações Complementares:



Para impossibilitar o cliente (já ativo nessa modalidade em outro momento) basta enviar valor 0 de limite de crédito.

Caso o distribuidor enviar duas requisições com valores diferentes para um cliente, o que prevalecerá como valor é sempre a requisição mais recente (os valores não são somados)

Enviar apenas atualizações, boas práticas, não repetir valores que não houve atualizações.
